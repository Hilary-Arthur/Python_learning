# 异常

## 了解异常
+ 当程序检测到一个错误时，Python解释器就无法继续执行了，反而出现了一些错误提示，即“异常 ”
## 异常捕获的办法（异常处理）
+ 基本语法：
  + ```
        try:
            可能发生错误的代码
        except:
            如果出现异常执行的代码
    ```
+ 捕获指定异常
  + 基本语法：
    + ```
          try:
              print(name)
          except NameError as e:
              print('name变量名称未定义错误')
      ```
+ 注意：
  + 如果尝试执行的代码的异常类型和要捕获的异常类型不一致，则无法捕获异常
  + 一般try下方只放一行尝试执行的代码
+ 捕获多个异常
  + 当捕获多个异常时，可以把要捕获的异常类型的名字，放到except后，并使用元组的方式进行书写
  + ```
        try:
            print(1/0)
        except (NameError,ZeroDivisionError):
            print('ZeroDivision错误...')
    ```
+ 捕获所有异常
+ 异常else
  + 表示如果没有异常要执行的代码
+ finally表示无论有没有异常都会执行
## 异常的传递
+ 当函数1发生异常但没有捕获处理时，异常会传递到函数2，当函数2没有捕获处理这个异常的时候main函数会补货这个异常，即异常具有传递性
+ 当所有函数都没有捕获异常时，程序就会报错
## Python模块
+ 模块的导入：
  + 导入方式[from 模块名] import [模块 | 类 | 变量 | 函数 | *] [as 别名]
  + 上述中括号表示可选
  + 具体案例见[Python模块.py](Python%E6%A8%A1%E5%9D%97.py)
+ 自定义模块
  + 导入不同模块的同名功能,后面一个会覆盖前一个的功能
+ 测试模块 
+ __main__变量
  + 当在该文件下执行的时候，__name__的值会变为__main__才会执行该语句下的内容
+ __all__变量
  + 如果一个模块文件中有__all__变量，当使用from xxx import * 导入时，只能导入这个列表中的元素
